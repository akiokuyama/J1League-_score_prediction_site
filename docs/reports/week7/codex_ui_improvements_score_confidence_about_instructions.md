# Codex指示書：予測説明・信頼度ラベル・過去予測サマリー改善

## 0. 位置づけ

この作業は、Streamlit β公開後に見つかったUI改善タスクです。
目的は、モデルやデータ更新処理を変更せず、ユーザーが予測結果を理解しやすくすることです。

今回対応する改善は以下の3点です。

1. 予測スコアと勝敗確率トップの違いを分かりやすく説明する
2. 勝敗確率トップの強さを示す信頼度ラベルを追加する
3. 過去予測結果画面の上部に精度サマリーを追加する

この作業はUI改善のみです。予測ロジック、モデル、データ取得、GitHub Actionsは変更しないでください。

---

## 1. 背景

現在のアプリでは、一覧カードや詳細画面で以下のような表示になる場合があります。

```text
予測スコア: 1-1
勝敗確率トップ: アウェイ勝利 51.2%
```

これはロジック上は矛盾ではありません。

- 予測スコアは、個別のスコア候補の中で最も有力な1パターン
- 勝敗確率トップは、ホーム勝利・引き分け・アウェイ勝利というカテゴリ単位の合算確率

そのため、個別スコアでは `1-1` が最有力でも、`0-1`、`1-2`、`0-2` など複数のアウェイ勝利スコアを合算すると、アウェイ勝利カテゴリが最も高くなることがあります。

この違いをユーザーが自然に理解できるように、表示上の補足を追加してください。

---

## 2. 対応タスクA：予測スコアと勝敗確率トップの違いを説明する

### 2.1 目的

ユーザーが「予測スコアは引き分けなのに、勝敗確率トップはホーム勝利/アウェイ勝利になっている」と感じたときに、違和感を持たないようにすることです。

### 2.2 実装方針

試合カード、または試合詳細画面に、短い補足説明を表示してください。

一覧カードでは情報量を増やしすぎないため、表示する場合は1行程度にしてください。
詳細画面には、もう少し丁寧な説明を表示して構いません。

### 2.3 表示例

予測スコアと勝敗確率トップの方向が一致している場合：

```text
予測スコアと勝敗確率トップは同じ方向を示しています。
```

予測スコアが引き分けで、勝敗確率トップがホーム勝利の場合：

```text
スコア候補では引き分けが最有力ですが、勝敗カテゴリ全体ではホーム勝利が最も高くなっています。
```

予測スコアが引き分けで、勝敗確率トップがアウェイ勝利の場合：

```text
スコア候補では引き分けが最有力ですが、勝敗カテゴリ全体ではアウェイ勝利が最も高くなっています。
```

予測スコアがホーム勝利で、勝敗確率トップが引き分けの場合：

```text
単一スコアではホーム勝利が最有力ですが、勝敗カテゴリ全体では引き分けが最も高くなっています。
```

予測スコアがアウェイ勝利で、勝敗確率トップが引き分けの場合：

```text
単一スコアではアウェイ勝利が最有力ですが、勝敗カテゴリ全体では引き分けが最も高くなっています。
```

### 2.4 推奨実装

可能であれば、表示ロジックを関数化してください。

例：

```python
def get_score_result_direction(predicted_score: dict) -> str:
    home = int(predicted_score.get("home", 0))
    away = int(predicted_score.get("away", 0))
    if home > away:
        return "home_win"
    if home < away:
        return "away_win"
    return "draw"


def get_probability_top_direction(result_probabilities: dict) -> str:
    keys = ["home_win", "draw", "away_win"]
    valid = {key: float(result_probabilities.get(key, 0.0) or 0.0) for key in keys}
    return max(valid, key=valid.get)


def build_score_probability_explanation(predicted_score: dict, result_probabilities: dict) -> str:
    score_direction = get_score_result_direction(predicted_score)
    prob_direction = get_probability_top_direction(result_probabilities)

    if score_direction == prob_direction:
        return "予測スコアと勝敗確率トップは同じ方向を示しています。"

    # ここで direction の組み合わせに応じた説明文を返す
```

既存の関数や構造がある場合は、それに合わせて自然に実装してください。

## 3. 対応タスクB：信頼度ラベルを追加する

### 3.1 目的

勝敗確率トップが同じ「ホーム優勢」でも、78%と42%では意味が違います。
この差を、ユーザーが直感的に理解できるようにしてください。

### 3.2 追加したい情報

既存の方向ラベルに加えて、確率の強さを示すラベルを表示してください。

既存または追加済みの方向ラベル：

- ホーム優勢
- アウェイ優勢
- 引き分け濃厚

今回追加する信頼度ラベル例：

- 確度高め
- やや優勢
- 拮抗

### 3.3 判定基準

まずは以下のルールで実装してください。

| 勝敗確率トップ | 信頼度ラベル |
|---:|---|
| 60%以上 | 確度高め |
| 45%以上60%未満 | やや優勢 |
| 45%未満 | 拮抗 |

`draw` がトップの場合は、方向ラベルは `引き分け濃厚` とし、信頼度ラベルは同じ基準で構いません。

例：

```text
引き分け濃厚 / やや優勢
```

または、見た目がごちゃつく場合は、以下のように1つの文にしても構いません。

```text
勝敗確率トップ：引き分け 49.6%（やや優勢）
```

### 3.4 UI方針

カード内にタグが多くなりすぎる場合は、以下のどちらかにしてください。

案A：タグを2つ表示

```text
ホーム優勢　確度高め
```

案B：既存の勝敗確率トップ表示に信頼度を追加

```text
勝敗確率トップ：ホーム勝利 78.6%（確度高め）
```

既存UIが見やすい方を優先してください。

### 3.5 色・テーマ対応

Light/Dark両方で読めるようにしてください。
文字色を薄くしすぎないでください。

既存のタグCSSがある場合は、それに合わせてください。
なければ、以下のようなテーマ対応を検討してください。


## 4. 対応タスクC：過去予測結果の精度サマリーを追加する

### 4.1 目的

過去予測結果画面の上部に、アプリの予測実績を一目で確認できるサマリーを表示してください。

### 4.2 表示したい項目

最低限、以下を表示してください。

- 評価対象試合数
- 勝敗的中率
- スコア完全的中率
- 直近5試合の勝敗的中数
- 直近5試合のスコア的中数

表示例：

```text
評価対象：20試合
勝敗的中率：55.0%
スコア完全的中率：15.0%
直近5試合：勝敗 3/5、スコア 1/5
```

### 4.3 集計対象

`outputs/past_prediction_results.json` の `matches` を対象にしてください。

すでに `app/utils/evaluation.py` に同様の関数がある場合は、それを再利用してください。
なければ、UI用の軽量関数を追加して構いません。

### 4.4 直近5試合の定義

可能であれば、試合日またはキックオフ日時で降順ソートし、直近5件を対象にしてください。
日付が欠損している場合は、元の並び順の末尾5件など、安全に処理してください。

アプリが落ちないことを優先してください。

### 4.5 データがない場合

過去予測結果が0件の場合は、以下のような表示にしてください。

```text
まだ評価対象の過去予測結果がありません。試合結果が反映されると、ここに的中率が表示されます。
```

エラーでアプリが落ちないようにしてください。

### 4.6 UI方針

過去予測結果画面のカード一覧より上に、KPIカードまたは横並びの小さなサマリーとして表示してください。

例：

```text
[評価対象 20試合] [勝敗的中率 55.0%] [スコア的中率 15.0%] [直近5試合 勝敗3/5・スコア1/5]
```

Streamlitの `st.columns` や既存のカードCSSを使って構いません。

Light/Dark両方で読めるようにしてください。

## 7. 実装上の注意

### 7.1 例外耐性

以下の場合でもアプリが落ちないようにしてください。

- `result_probabilities` が欠損している
- `predicted_score` が欠損している
- `past_prediction_results.json` が存在しない
- `matches` が空
- 過去予測結果に日付がない
- 数値が文字列で入っている

### 7.2 Light/Darkモード対応

今回追加する説明文、タグ、サマリーカードは、Light/Dark両方で読めるようにしてください。

特に以下に注意してください。

- 文字色を白系や薄いグレーに固定しない
- 背景色だけ薄くして文字が読めなくなる状態にしない
- 既存のタグ修正方針と統一する

### 7.3 表示の出しすぎに注意

一覧カードに長文説明を入れすぎると見づらくなります。
一覧では短く、詳細画面では少し丁寧に説明してください。

## 8. 確認観点

### 8.1 予測説明

- 予測スコアと勝敗確率トップが一致している試合で、自然な説明が表示される
- 予測スコアと勝敗確率トップが異なる試合で、違いが分かる説明が表示される
- 表示文が長すぎない
- Light/Dark両方で読める

### 8.2 信頼度ラベル

- 60%以上で `確度高め` が表示される
- 45%以上60%未満で `やや優勢` が表示される
- 45%未満で `拮抗` が表示される
- ホーム勝利・アウェイ勝利・引き分けのすべてで動作する
- Light/Dark両方で読める

### 8.3 過去予測サマリー

- 評価対象試合数が表示される
- 勝敗的中率が表示される
- スコア完全的中率が表示される
- 直近5試合の的中数が表示される
- 過去予測結果が0件でもアプリが落ちない
- 過去予測カード一覧の表示が壊れていない
