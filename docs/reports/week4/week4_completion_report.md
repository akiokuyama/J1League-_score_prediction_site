# Week4 完了レポート

## 概要

Week4では、Week3までに生成できるようになった予測結果JSONを、ユーザーがブラウザ上で確認できる **Streamlit MVP** として実装しました。

今回の主目的は、データ取得・特徴量生成・モデル推論の追加ではなく、既に生成済みの予測結果を見やすく表示するアプリ画面を作成することです。

実装後、ローカル環境で Streamlit アプリが起動できること、実データを読み込んで表示できること、過去予測結果の確認画面が動作すること、既存テストが通ることを確認しました。

## 実施した作業

- `outputs/latest_predictions.json` を読み込む Streamlit MVP を実装
- 「これからの試合」と「過去の予測結果」を切り替えられる画面を実装
- モバイル表示を意識し、「試合一覧 → 試合詳細」の画面遷移を実装
- 試合カード全体をクリックすると詳細画面へ遷移できるように実装
- 詳細画面から `← 試合一覧に戻る` で一覧に戻れるように実装
- 勝敗確率トップと予測スコアを分けて表示
- 予測スコアと勝敗確率トップがズレる場合の説明文を表示
- 得点者候補Top5を折りたたみ表示に変更
- 「ホーム優勢」「アウェイ優勢」の見立てラベルを表示
- これからの試合の絞り込み機能を実装
- 過去予測結果の表示画面を実装
- 過去予測について、勝敗判定とスコア判定を別々に表示
- 過去予測の勝敗的中率・スコア的中率を表示
- 過去予測の節・チーム・判定結果による絞り込みを実装
- 空データ、欠損キー、壊れたJSONでもアプリが落ちないように防御的な処理を追加
- `matchweek` / `section` が欠損している過去予測データでも節フィルタで落ちないように `safe_int()` を追加

## 画面構成

### 1. これからの試合

`outputs/latest_predictions.json` を読み込み、次節の予測対象試合を一覧表示します。

主な表示項目は以下です。

- 試合日
- 会場
- ホームチーム / アウェイチーム
- 予測スコア
- 勝敗確率トップ
- 見立てラベル
  - ホーム優勢
  - アウェイ優勢
- 詳細画面への遷移

一覧画面では、以下の絞り込みに対応しています。

- チーム
- 試合が行われる節

### 2. 試合詳細

試合カードをクリックすると、対象試合の詳細を表示します。

主な表示項目は以下です。

- 予測スコア
- 結論カード
- 期待得点
- 勝敗確率
- 勝敗確率の注意書き
- スコア候補Top5
- 得点者候補Top5

得点者候補は情報量が多いため、初期表示では折りたたみ表示にしています。

### 3. 過去の予測結果

`outputs/past_prediction_results.json` が存在する場合、過去に出した予測と実際の試合結果を並べて表示します。

主な表示項目は以下です。

- 予測スコア
- 実際のスコア
- 勝敗的中 / 勝敗外れ
- スコア的中 / スコア外れ
- 予測の勝敗方向
- 実際の勝敗方向
- 勝敗的中率
- スコア的中率
- 表示試合数

過去予測画面では、以下の絞り込みに対応しています。

- チーム
- 試合が行われた節
- 予測結果に対する判定

## 勝敗確率と予測スコアの扱い

Week4では、ユーザーが混乱しやすい以下の点について表示を改善しました。

- 予測スコアは、単一のスコア候補として最も有力なもの
- 勝敗確率トップは、ホーム勝利・引き分け・アウェイ勝利という結果カテゴリごとの合算値

そのため、たとえば予測スコアが `1-1` でも、`0-1`、`1-2`、`0-2` など複数のアウェイ勝利スコア候補を合算すると、アウェイ勝利の確率が最も高くなる場合があります。

このようなケースでは、詳細画面に補足説明を表示するようにしました。
