# 天候特徴量の採否レポート

## 結論

正式モデルでは、天候特徴量を使用しない方針を採用しました。

理由は、天候なしモデルで得点MAEがほぼ横ばい、勝敗AccuracyとLogLossが改善し、最終予測の性能も大きく崩れなかったためです。Goal Diff MAEは悪化しましたが、天候データ取得・更新の運用負荷と欠損リスクを考えると、正式パイプラインでは除外する判断が妥当です。

## 評価条件

- データ: `Data/ML_dataset.csv`
- 学習データ: `Season < 2025`
- テストデータ: `Season == 2025`
- 天候あり: `Weather` と `Weather_*` を含む既存方針
- 天候なし: `Weather` を除外し、One-Hot対象を `Backline_Matchup`、`Home_Formation`、`Away_Formation` に限定

## 比較結果

| 指標 | 天候あり | 天候なし | 差分（天候なし - 天候あり） |
|---|---:|---:|---:|
| Home MAE | 0.8991 | 0.8992 | +0.0001 |
| Away MAE | 0.8037 | 0.8038 | +0.0001 |
| 平均MAE | 0.8514 | 0.8515 | +0.0001 |
| 勝敗Accuracy | 0.4921 | 0.5079 | +0.0158 |
| 勝敗LogLoss | 1.0754 | 1.0589 | -0.0165 |
| Goal Diff MAE | 1.0638 | 1.0893 | +0.0255 |
| 最終スコアEMR | 0.1763 | 0.1737 | -0.0026 |
| 最終Home MAE | 0.9132 | 0.9132 | +0.0000 |
| 最終Away MAE | 0.8421 | 0.8447 | +0.0026 |
| 最終勝敗Accuracy | 0.4921 | 0.4974 | +0.0053 |

## 採用判断

採用基準に対する判定は以下です。

| 観点 | 判定 |
|---|---|
| 平均MAEの悪化 +0.01 未満 | 達成 |
| 勝敗Accuracyの低下 0.5pt 未満 | 達成。低下ではなく改善 |
| 勝敗LogLossの悪化 +0.01 前後まで | 達成。悪化ではなく改善 |
| Goal Diff MAEの悪化 +0.01 未満 | 未達。+0.0255 悪化 |
| 最終スコアEMRが同等または改善 | ほぼ同等。-0.0026 |

Goal Diff MAEのみ基準未達ですが、得点予測・勝敗予測・最終予測の主要指標は同等以上であり、天候取得の難易度と運用上の不安定さを避ける効果が大きいため、天候なしモデルを正式採用しました。

## 実装方針

- `Weather` は学習・推論の入力から除外
- `Weather_*` のOne-Hot特徴量は生成しない
- `Models/model_features.pkl` は天候なしの158特徴量で更新
- `Models/feature_policy.json` に天候除外方針を保存
- `src/predict/feature_preprocess.py` でも天候列を入力から落とす

## 今後の注意

今後、天候データを再検討する場合は、試合前時点で取得可能な予報データだけを使い、取得失敗率・更新遅延・予測時点の情報リークを同時に評価してください。現時点では正式特徴量へ戻す必要はありません。
